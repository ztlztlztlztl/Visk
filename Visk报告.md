## 一、 程序功能介绍

**Visk**是一款高性能、高颜值文件管理工具，在保留Windows原生文件管理器核心能力的基上，引入了 **可视化的磁盘内存图形化展示功能** 与创新的 **“文件岛”** 交互功能，下面是核心功能介绍：

- **1，设备磁盘读取**：Visk可以扫描本地设备，直接读取到已激活的磁盘的名称、盘符、内存使用情况等信息，包括外部接入的U盘等。
- **2，快速内存扫描**：Visk可以对选定的磁盘进行快速内存扫描，在短时间内可以获取这个磁盘下所有文件夹及文件信息，并构建文件树，从而实现列表展示、视图展示等一系列功能。
- **3，两种文件视图**：对于已经扫描好的磁盘，Visk支持朴素的列表视图展示和图形化视图展示。
  - **朴素的列表视图展示**就是像Windows原生的文件管理器一样，每个文件夹/文件条状展示，并展示出名称、大小、类型、修改时间这四个基本信息，并支持按这四个标签分别进行排序。
  - **图形化视图展示**就是以文件夹/文件的内存大小为基准，通过算法对该目录下所有文件夹/文件生成一个图形，图形中不同的矩形代表不同的文件夹/文件，且矩阵大小象征内存大小，使得内存占用可视化。
  - 同时这两个视图都支持Windows原生文件管理器的**双击下沉、双击打开、右键属性查看、顶部路径导览**的基本功能。
- **4，关键词搜索**：在搜索视图中，可以键入关键词，选择搜索范围，可以实现目标文件的秒搜，并且搜索出的列表可以通过双击进入朴素的列表视图展示，同样有双击下沉、双击打开、右键属性查看功能。
- **5，创新功能“文件岛”**：为解决原生文件管理器对文件处理**自由度低**的痛点，引入了暂存式的交互理念，从而得出 **“文件岛”** 这一概念。用户可通过**拖拽**将**跨目录**的文件夹/文件集中到屏幕底部的“文件岛”中，可以通过简单操作统一执行**批量复制、移动、删除、一键改后缀或注入系统剪贴板**等操作，也可以实现原生的单文件重命名，极大提升了**跨目录文件处理的效率**。
- **6，整体的清新气泡感UI界面**：页面通过整体上采用 **圆角 + 多巴胺配色 + 悬浮高亮** 的界面设计，使得整体上颜色柔和养眼又不失清晰，**交互体验**极好。

### 前端

按UI元件分成三部分来讲。

#### 1，磁盘列表(drivelist)

- **概述**：类`driveListZone`提供了简单的磁盘信息展示，并作为内存扫描启动按钮。
- **信息获取**：通过调用**Windows操作系统的API**来快速获取当前设备上的磁盘，开机**自动**扫描一次。
- **信息储存**：通过一个简单结构体`DriveInfo`来记录一个磁盘的基本信息，即：
    ``` C++
    struct DriveInfo{
        QString letter;     // 盘符
        QString name;       // 磁盘名
        QString filesystem; // 磁盘格式
        qint64 totalBytes;  // 总空间
        qint64 freeBytes;   // 可用空间
        bool isNtfs;        // 磁盘格式是否是 NTFS
    };
    ```
- **信息展示**：类`DriveCardWidget`构建为简单的卡片样式，排列展示，点击卡片即可对该磁盘进行扫描。
- **辅助功能**：加入全局磁盘重扫、单磁盘重扫功能，增加程序鲁棒性。

#### 2，文件展示(filedisplayer)

- **基本构成**：路径导览条、列表视图、图形化视图、搜索视图。
- **底层信息流**：在接收到**磁盘列表**发出的信号后，从类`general_control`中获得文件信息，然后将所有文件信息储存入数据模型类`fileDisplayModel`中，之后绘制类`tableStyleWidget`与`treemapStyleWidget`将会从模型中获取信息进行绘制。其中类`treemapStyleWidget`中包含了后端的图形分割算法。
- **文件展示系统**：因为在绘制类`tableStyleWidget`与`treemapStyleWidget`中重载了双击与右键的函数，从而实现了双击下沉和右键属性的功能，功能实现均依赖信号与槽机制，在`mainwindow.cpp`中处理从而维护了路径导览条的位置一致性。
- **路径导览条**：路径导览条类`breadcrumbWidget`通过多个`button`拼接而成，故完美实现了点击回退的功能。
- **搜索视图**：通过简单的输入框和下拉栏获取信息，并用按钮启动，像列表视图的处理一样，通过数据模型类`fileDisplayModel`和绘制类`tableStyleWidget`实现。
- **拖拽功能**：三视图均重载了拖动函数，拖动信息为精简的`file_location`，故可以实现文件拖出功能，方便下面的文件岛功能。

#### 3，文件岛(fileisland)

- **基本构成**：文件岛是一个大类`fileIslandWidget`，有内部许多元件拼成，从左到右有三个隐式分区：**Object, Action, Destination**。
- **Object**：该区域是一个`QList`，内部是一个个代表文件的`Label`，通过接受**拖拽**添加文件，并带有删除按钮。
- **Action**：该区域由多个按钮**堆叠**而成，通过组合为一个**按钮组**实现只能单一选中，每个按钮绑定一个信号，用于呼出对应的**Destination页**。
- **Destination**：该区域是一个`QStackedWidget`与**多个页面**和一个**Do**按钮组合而成，每个页面对应一个action按钮，点击do按钮即执行操作。
- **实现说明**：**Do**按钮平常是**禁用**的，每当有变化时会**自检**是否合法，合法后亮起；移动与复制操作在**选择目标路径**时，启动一个选择窗口，通过复用文件展示中的列表视图来实现同盘选择；**底层**均通过调用类`general_control`中的接口。
- **底部抽屉**：整个文件岛通过放在`QDockWidget`中，用一个**按钮**实现**隐藏与显示**。

#### 4，QSS样式(constants)

绝大部分元件的样式通过**QSS样式表**实现美化，即**圆角+多巴胺配色+悬浮高亮**显示，这样的处理使得界面更加**美观清晰**，并且增加了**交互感**。

#### 5，其余窗口

- **报错窗口**：报错窗口连接到信号`operation_error`，在出错时简单启动`QMessageBox`来表示出错。
- **属性窗口**：属性窗口连接到信号`requestShowProperty`，要表示属性时常见一个类`PropertyDialog`，显示一个文件夹/文件的基本信息。
